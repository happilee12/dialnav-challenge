The 1st DialNav Challenge

Hosted at the Embodied Agent and Dialog (EAD) Workshop, ECCV 2026.

Overview

The DialNav Challenge benchmarks embodied agents that follow high-level, intentionally ambiguous natural-language instructions in indoor environments. Successful agents must combine vision-and-language navigation with proactive dialog: detecting uncertainty, asking clarifying questions, and updating their beliefs as the conversation progresses.

The challenge contributes a benchmark, a starter kit, and an evaluation server. Top-performing teams will be invited to present at the EAD Workshop at ECCV 2026.

Task

DialNav couples a Navigator (the embodied agent that moves through the scene) with a Guide (the dialog partner that holds the goal information). Both interact via natural language. Participants may modify the Navigator, the Guide, or both, and submit the resulting system for evaluation.

What We Provide

  • RAIN dataset — episodes, instructions, and reference dialogs.
  • Starter code — data loaders, evaluation scripts, submission tooling.
  • Navigator and Guide reference implementations.
  • Pretrained weights for the baseline Navigator and Guide.
  • Baseline Docker image reproducing the reference setup.
  • Sample episodes with open Navigator and Guide trajectories.

Rules & Constraints

Hardware

  • 1 × NVIDIA A100 40GB or 4 × RTX 3090 (or equivalent).
  • Up to 16 GB CPU RAM.

Per-episode limits

  • Maximum runtime: 5 minutes per episode.
  • Maximum steps: 50 per episode.

Submissions exceeding any of the above limits will be marked as failed for the affected episodes.

Submission

Each submission must include:

  • Model code (Navigator and/or Guide).
  • Model weights required to reproduce the run.
  • Execution logs on the validation set.
  • A short technical report describing the method.

Submissions are made through the challenge submission portal (link will be provided at launch). A reproducible Docker image is recommended.

Evaluation Metrics

  • SR — Success Rate.
  • SPL — Success weighted by Path Length.
  • NE — Navigation Error (final distance to the goal).
  • Dialog Turns — number of dialog turns per episode.

Leaderboard

The leaderboard is operated on the validation set, based on execution log files uploaded by participants. All submitted log files are made publicly available for transparency and reproducibility.

Because validation ground truth is public, log-file ranking alone is susceptible to manipulation. To address this, the organizers re-evaluate the submitted code on a held-out portion of the test set for the final ranking. Teams targeted for the final leaderboard must therefore submit reproducible code in addition to their logs.
